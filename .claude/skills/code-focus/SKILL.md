# Code Focus Skill

## 核心原则

**只做必要的代码修改，不要过度解读或生成文档。**

## 指令

### 代码修改原则
- **只修复问题描述中明确提到的内容**
- **不要添加额外的功能或"改进"**
- **不要生成 README、注释、文档等，除非明确要求**
- **保持代码修改最小化，只改必要的地方**

### 避免过度解读
- **严格按照问题描述执行，不要推测额外的需求**
- **如果问题描述不明确，只做最保守的修复**
- **不要添加"防御性"代码或"最佳实践"，除非明确要求**
- **不要重构代码，除非明确要求重构**

### 文档生成限制
- **禁止自动生成或更新 README.md**
- **禁止自动生成或更新 CHANGELOG.md**
- **禁止添加长篇注释或文档字符串（除非明确要求）**
- **代码注释应该简洁，只解释"为什么"而不解释"是什么"**

### 上下文限制
- **只读取和修改与问题直接相关的文件**
- **不要探索整个代码库寻找"潜在问题"**
- **不要添加新的依赖或工具，除非明确要求**

### 输出限制
- **提交信息应该简洁，只描述做了什么**
- **不要生成冗长的 PR 描述**
- **不要在代码中添加 TODO 或 FIXME，除非明确要求**

## 示例

✅ **正确做法：**
- 问题："修复登录按钮样式"
- 行动：只修改登录按钮的 CSS，不修改其他按钮

❌ **错误做法：**
- 问题："修复登录按钮样式"
- 行动：修改所有按钮样式，添加 README 说明，重构按钮组件
