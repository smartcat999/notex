# Minimal Changes Skill

## 核心原则

**最小化修改，只改必要的地方。**

## 指令

### 修改范围
- **只修改与问题直接相关的代码行**
- **不要修改同一文件中其他不相关的代码**
- **不要重构整个函数，只修改出错的部分**
- **保持原有的代码风格和结构**

### 文件修改
- **只修改必要的文件**
- **不要创建新文件，除非明确要求**
- **不要删除文件，除非明确要求**
- **不要移动文件，除非明确要求**

### 依赖管理
- **不要添加新的依赖包**
- **不要升级依赖版本，除非明确要求**
- **不要修改 package.json、requirements.txt 等，除非明确要求**

### 测试修改
- **不要添加测试，除非明确要求**
- **只修改失败的测试，使其通过**
- **不要重构测试代码**

### 配置修改
- **不要修改配置文件，除非明确要求**
- **不要添加新的配置项**
- **不要修改环境变量或部署配置**
